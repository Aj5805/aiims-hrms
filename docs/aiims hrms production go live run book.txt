## 0. Pre-go-live decisions (go / no-go)

Decide each consciously. These are known launch constraints, not newly found bugs:

- [ ] Email notifications are still a stub. `email_sender.start_email_scheduler` is not started, so approval/notification emails will not send. Accept for launch, or wire real Zoho SMTP first. If enabled, it must run in exactly one worker; the current `--workers 4` service shape would duplicate the scheduler.
- [ ] 5.6 resident pro-rata recompute is not implemented. There is still no contract-lifecycle event model or endpoint to recompute resident balances on contract completion/extension. Acceptable only if resident contracts are stable at launch.
- [ ] Leave ledger is still schema-derived, not a dedicated immutable leave-transaction table. The `/leave-balances/:employee_id/ledger/:leave_type_id` endpoint now combines opening, annual credit, manual adjust, and approved/recalled application events, but this is still derived from current schema.
- [ ] Preview-before-commit for year-end processing is not implemented. The admin screen can run carry-forward, annual credit, and manual adjust actions, but does not yet show a preview simulation step.
- [ ] Git tags do not exist yet. `ROLLBACK.md` references release tags that still need to be created during cutover.
- [ ] Confirm the real access hostname (for example `hrms.aiims-bibinagar.local` or a fixed server IP). It is needed for the cert CN, `CORS_ORIGINS`, and UAT URLs.

## 6. Backend as a service (NSSM)

```powershell
nssm install AIIMS_HRMS "C:\Python311\python.exe" "-m uvicorn main:app --host 127.0.0.1 --port 8000 --workers 4"
nssm set AIIMS_HRMS AppDirectory "C:\aiims-hrms\backend"
nssm set AIIMS_HRMS AppStdout "C:\aiims-hrms\logs\uvicorn_stdout.log"
nssm set AIIMS_HRMS AppStderr "C:\aiims-hrms\logs\uvicorn_stderr.log"
nssm set AIIMS_HRMS AppRestartDelay 5000
nssm set AIIMS_HRMS Start SERVICE_AUTO_START
nssm start AIIMS_HRMS
nssm status AIIMS_HRMS
```
